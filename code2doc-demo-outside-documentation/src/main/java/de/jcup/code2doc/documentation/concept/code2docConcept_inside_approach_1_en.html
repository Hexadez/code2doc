The inside approach has following advantages:

In your Code

You can reference all your project code parts to
  • UseCases via annotation @LinkToUseCase
  • Architectures via annotation @LinkToArchitecture
  • Roles via annotation @LinkToRole
  • Constraints via annotation @LinkToConstraint

In your IDE

  • Search for dependencies for the linked part - so e.g. you know exactly which classes, URLs etc. belong to use cases
  • Search for use cases by type hierarchy
  • Refactor your documentation automatically via your IDE tools
  • Automatical specification building. No manual definitinos are necessary (except the element definitions).
