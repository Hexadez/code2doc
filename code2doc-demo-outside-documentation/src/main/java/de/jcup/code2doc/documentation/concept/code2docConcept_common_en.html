When you want to use code2doc you must always create a "documentation" project. You also should define another project "documentation-output"

In the "documentation" project

You can define

  • Concepts
  • UseCases
  • Architectures
  • Roles
  • Constraints
Use a package structure when you define classes.

In the "documentation-output" project

Create a class with a main method where you can start the output generation. There must be a specification created and given to a dedicated output renderer.
